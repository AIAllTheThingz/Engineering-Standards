## Schema Impact

No breaking schema changes are introduced by publication. Existing `1.0.0` schema contracts remain supported, and additive `1.1.0` evidence semantics are used where the repository already supports them.

## Workflow And Action Impact

No workflow or action implementation is changed by this publication documentation update. Downstream repositories should continue to pin reusable workflow calls to immutable commit SHAs.

## Migration

Downstream repositories may adopt `1.1.0` at normal cadence if they already satisfy the `1.0.0` schema contracts. Integration-heavy repositories should review the strengthened Integration standard before making it blocking.

Workflow callers should update `governance-version` to `1.1.0` when their `project-manifest.json` also records `governanceVersion` as `1.1.0`.

## Downstream Guidance

- Production downstream repositories should pin reusable workflows to immutable commit SHAs.
- Teams using copied governance files should replace them with versioned central references or document the controlled copy and update path.
- Repositories adopting the Integration standard should validate webhook, queue, file-transfer, retry, idempotency, partial-success, and evidence behavior before enforcing it.
